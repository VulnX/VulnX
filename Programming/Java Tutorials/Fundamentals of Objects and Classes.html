Introduction

In Java, every line of code is written within a class. A class serves as a template for creating similar kind of objects.

As of now you can understand that class is a container for all data [ variables ] and code [ functions ] & an object is a reference to a class.

Basics - Object

An object is the most basic unit of an Object Oriented Programming (OOP) language. In Procedure Oriented Programming (POP) languages, the whole code is written at a single place without enclosing it in any function or class. Like the example given below :-

section .text

global init

init:

    push 0x1
    pop rax
    mov rdi, rax
    mov rsi, message
    mov rdx, length
    syscall

    push 0x3c
    pop rax
    xor rdi, rdi
    syscall

message db "Test", 0xa
length equ $-message
The above code is not of Java so it's not necessary to understand it.

What you can see above is probably the simplest x64 NASM assembly code [ INTEL SYNTAX ]. Understanding the code isn't important because this is not Java so it's better to not focus on it. But the reason I've included it here is that, assembly is a POP language and you can see above, how everything is just written openly and not enclosed in any function or class.

What you see above as 'init', 'message' and so on, are not function and data but rather just labels for the linker to process information better.

Thus unlike procedural languages, in OOP attempts are made to split a task into multiple classes.

Message Passing

Remember in the last tutorial, we had learnt about Encapsulation, there we had understood that only the functions can communicate with each other and by no means can a function access the private data of another object. This is true, however in reality, it is not the function which directly communicates with the other functions, it is actually the objects with interact with each other, with the concept of Message Passing, allowing functions to call other functions and in turn allow them to use only the esssential data.

*Defination:
Message Passing is the concept of interaction of objects with each other, allowing them to share and accept information for the better overall functioning of the program.

A showroom of cars is a representation of a class

Writing your first Java Program

Let's see how we can create a code to *print a line in Java.

*print - here does not mean physically printing a line on paper, it means "displaying" a text sentence on the console.

The SYNTAX for printing a line on the console is as follows :

System.out.println("<TYPE HERE THE SENTENCE TO PRINT>");

For example :

System.out.println("Hello World!");

Note the SYNTAX properly.

SYNTAX Note
System Keep in mind to write the 'S' in uppercase while the rest of the word in lowercase.
We use this to access the System library and it's other built-in functions.
out The whole word will be in lowercase.
This is an object in the System library, we call this object in order to tell the computer to prepare for outputting something.
println println is a built-in system function so be sure to write it in this format ONLY.
We call this function to tell the computer to print [ in normal english, 'display' ] something on the console screen.
("MESSAGE") Remember, we had read earlier in this tutorial that a () indicate a function?
Since println is a function, it will be followed by these () , and the content to be printed is placed within these parenthesis.
"MESSAGE" Since the message we have to display to the console is a String [ basically a sentence, we will learn about Strings later ], we need to enclose it within the ""
If not done so, the computer won't be able to distinguish between a sentence and a variable*.
MESSAGE This is the text to print, so you can replace it to whatever you want. So no restriction on anything :)
; This is what most people miss!
Often they forget to add this ; (semi-colon) in the end and the program fails to compile.
The ; (semi-colon) is a line-terminator, every Java statement should end with a ; (semi-colon).
. Make sure to put a . (period) between System, out & println

*For example you create a program to calculate the sum of a few numbers, then you store the sum of all numbers in a variable called sum to print it later on. Now if you print without the "" it will be like :
System.out.println( sum = + sum );
This doesn't make any sense even to the normal english, how will the computer be able to understand the difference between the sum?
So to avoid this confusion, we need to enclose it within "" .

Now write this line in the editor and try compiling it.

Did it work?
Obviously NOT! Reason?

The reason behind this is that, you can't simply write a line anywhere and expect the computer to execute it randomly. Every statement to be executed must be within a function and a function must be placed within a class. But the question here is that, in which function should we place this statement? In order to execute a statement, we need to place it within a function, but even that function won't execute until it is called. So the solution is, we need to place it in such a function that is called automatically by the computer when the program loads. Luckily we have a function called main

So the complete structure of a basic Java program will be as follows

class ClassName {
    public static void main(String[] args) {
        // Statements here
    }
}
ClassName can be changed in your case. It is not a keyword.

Rules to be kept in mind while naming a class

  • A class name should preferable always begin with an upper-case character.
  • A class name must not contain whitespace.
  • If more than one word has to be used in the class name then it can be done like this :
    FirstSecond
    First_Second
    • Underscore(_) is used to separate the words and not minus sign(-).
  • A class name may be a mixed-case alphanumeric word.
  • A class name cannot start from a digit like: 1501abcClass.
  • No special symbols are permitted apart from underscore(_).
public static void main(String[] args) cannot be changed. Each individual word of this statement will be explained in future tutorials for better understanding.

So let's write the final program to print "Hello World"

class Program {
    public static void main(String[] args) {
        // Call the println function through the out object of the System library.
        System.out.println("Hello World!");
    }
}

Now you compile the program. The compiling process might differ between different IDEs, but I am using the command-line interface so I will compile it like this

$VulnX ❯ javac Program.java
[+]    No Compilation Errors!

Great, no errors in our program & it's successfully compiled. Now let's run our application & test if all went correct.

$VulnX ❯ java Program
Hello World!
