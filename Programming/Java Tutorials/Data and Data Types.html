Need for Data Types

Essentially data types are, special types of data which tells the compiler or the interpreter how the programmer intends to use the data .

For example,
the sentence : "Hello, how are you?" and the number : 1501 are both, data to a computer. However it is essential for them to be separated.

The main reason to store these data into different categories like string for sentences and int for numbers, is to avoid mixing up of data. Let's say you write a program to accept 10 numbers from the user as input, and do some complex mathematical calculations on them to find some special number. But when the user actually runs the program and types a sentence instead of a number, your entire calculation system will go wrong and the program will fail. Data Types in Java

In the simplest terms, you can say that, data types are very specific containers used to store a particular type of data.

Now let's understand how many types of Data Types are there in Java.

Most distinctly, there are 2 broad classifications for Data Types. They are as follows:

  1. Primitive Data Types are the basic data types which are pre-built in the system.
  2. Composite Data Types are the user-defined data types which aren't provided by default by the system.

So here is a very brief distinguising table between Primitive and Composite data types.

Primitive Data Types Composite Data Types
Created by the system, i.e inbuilt. Created by the user, i.e user-defined.
For example:
int ; float ; long ; byte		 For example:
arrays ; classes ; interfaces

Since COMPOSITE data types are relatively complex, we won't learn about them so early. It's better to first understand the basics of Java and then explore the advanced concepts.

Primitive Data Types

As discussed earlier, primitive data types are provided by the computer.

There are 8 primitive data types in Java. They are:-

  1. byte
  2. short
  3. int
  4. long
  5. float
  6. double
  7. boolean
  8. char

Each of these, are capable of storing one particular type of data. For example, boolean can only store the values true or false. Storing any value apart from these 2, as a boolean data type will raise an error.

Each data type, is created with a specific amount of size. For example, the int data type has a memory storage capacity of 4 bytes. First, let's understand how computers measure the amount of memory.

  • Computers store data in the form of binary numbers...
  • ...which comprises of only zeros(0) and ones(1)
  • The most basic (smallest) unit in which computers measure data is called bit.
  • 1 Bit is equivalent to a 0 or 1.
  • This 0 or 1 is the same as turning off or on of a transistor inside your computer's memory.

Now let's see the bigger units of measurement which are more commanly used.

  1. Nibble (not very common)
  2. Byte (b)
  3. Kilobyte (KB)
  4. Megabyte (MB)
  5. Gigabyte (GB)
  6. Terabyte (TB)
  7. Petabyte (PB)
  8. Exabyte (EB)
  9. Zetabyte (ZB)
  10. Yotabyte (YB)
The underline units of measurement are most commanly used. The remaining ones are usually very small and very high.

Now let's see the relationship between these data sizes.

4 bits                     =    1 Nibble
8 bits or 2 nibbles        =    1 byte (b)
1024 bytes (b)             =    1 Kilobyte (KB)
1024 Kilobytes (KB)        =    1 Megabyte (MB)
1024 Megabytes (MB)        =    1 Gigabytes (GB)
1024 Gigabytes (GB)        =    1 Terabyte (TB)
1024 Terabytes (TB)        =    1 Petabyte (PB)
1024 Petabytes (PB)        =    1 Exabyte (EB)
1024 Exabytes (EB)         =    1 Zetabyte (ZB)
1024 Zetabytes (ZB)        =    1 Yotabyte (YB)

So below, I have presented a brief table showing all data types along with their sizes, range, data ( what type of data they are used to contain ) and default value.


DATA TYPE DATA SIZE RANGE DEFAULT VALUE
boolean true | false 1 bit =[NONE]= false
byte Small numeric values 1 byte -128 to +127 0
char Character 2 bytes =[NONE]= \u0000
short Small numeric values 2 bytes -32768 to +32767 0
int Numeric values 4 bytes -2147483648 to +2147483647 0
long Large numeric values 8 bytes -9223372036854775808 to +9223372036854775807 0
float Floating-point (decimal) values 4 bytes 1.4E-45 to 3.4028235E38 0.0f
double Large and precise floating-poing values 8 bytes 4.9E-324 to 1.79E308 0.0d

“byte” is both, a unit of data measurement as well as data type.

The SYNTAX for creating a variable with a data type is as follows:-

<DATA_TYPE> <VARIABLE_NAME> = <VALUE>;

OR

<DATA_TYPE> <VARIABLE_NAME>;
<VARIABLE_NAME> = <VALUE>;
The data type will only be used while creating a variable. Later if its value needs to be changed it must not include the data type as demonstrated in the second example, above. Thus the code like: 
<DATA_TYPE> <VARIABLE_NAME>;
<DATA_TYPE> <VARIABLE_NAME> = <VALUE>;
will be wrong. The correct one should be : 
<DATA_TYPE> <VARIABLE_NAME>;
<VARIABLE_NAME> = <VALUE>;
or 
<DATA_TYPE> <VARIABLE_NAME> = <VALUE>;

Now we will learn about all these data types in detail with practical programming examples.

1. boolean

Boolean is a special data type. Unlike other data types which store numbers or words/alphabets, boolean is used to store the values : true or false.

A variable declared with this data type, uses 1 bit of the computer's memory.

In a Java program we would declare a boolean variable as follows:

boolean my_variable = true;

We can create multiple variables of the same data type and they may or may not have the same value. Just be sure to name that new variable differently. As 1 or more variables with the same name will raise an error.

boolean variable1 = true;
        boolean variable2 = false;

Now let's add a few print statements to verify what's the answer

boolean variable1 = true;
        boolean variable2 = false;
        System.out.println(variable1);
        System.out.println(variable2);

The answer, quite noticeably is:

true
false

This is mostly used for conditional checking. In a situation where you need to execute some code depending on some condition, at that time we can use these boolean variables to store the outcome of our condition and later check if our condition is satisfied or not.

2. byte

The byte data type occupies 1 byte or 8 bits of the computer's memory.

It is used to store non-floating ( non-decimal i.e. integer ) numbers ranging from -128 to +127.

Let's assume we have a program which shows multiple options, each labelled with a number as shown below:

System.out.println("1. ABC");
        System.out.println("2. PQR");
        System.out.println("3. DKC");
        System.out.println("4. XYZ");
The output will be : 
1. ABC
2. PQR
3. DKC
4. XYZ
I won't be providing output everytime like this. It's assumed that so far you are clear with the conepts and can predict the output.

Suppose, after this, we somehow ask the user for input* and s/he will need to enter a number from 1 to 4 and accordingly we will do the rest of the processing.

* We will learn about user-input in the upcoming lessons.

Most people tend to store the user-input in a variable with the data type int. However, I believe that we should preferably use byte in these cases.

Both byte and int can store a number from 1 to 4 as it falls in their range capacity ( check the table given above to see the range capacity of all the data types ).

But int data type uses 4 bytes of the computer's memory while byte data type only uses 1 byte of the computer's memory.

So why not use byte if it can do the job and also use 4x less memory.

Here's how to make a byte variable

byte b = -12;
        System.out.println(b);

OUTPUT :-

-12
Moving a value which is beyond the data type's range capacity will raise an error. Like this: 
byte var1 = -1501;
        byte var2 = 1103;
        byte var3 = 0;
        System.out.println(var1);
        System.out.println(var2);
        System.out.println(var3);

It raises an error in the first 2 lines of the main method while compiling:

Program.java:3: error: incompatible types: possible lossy conversion from int to byte
        byte var1 = -1501;
                    ^
Program.java:4: error: incompatible types: possible lossy conversion from int to byte
        byte var2 = 1103;
                    ^

So be sure to use the correct data type according to your data.

3. char

The char is a data type used to store characteracters.

A character can be:

  • An alphabet [ lower case or upper case ] like, 'a' or 'Z'.
  • A digit like '1', '2', '5' or '4' and NOT 343.
  • An operator like '+', '-', '%', '^', '&', '*', '='.
  • All special symbols* like !, @, #, $, (, ), _, [, ], ;, :, ?, . and so on.
* Special symbols are basically everything except an alphabet, digit & operator in a sentence.

I hope you get the gist now.

Each character like 'A' or '%' or '6' is of 1 byte. However, the char data type is of 2 bytes. This is because Java uses UTF-16* encoding system.

* Read more about UTF-16 encoding system here.

Here's how you can use it in your program

char ch = 'X';
        System.out.println(ch);

OUTPUT :-

X
You can also directly write the value in the print statement like this: 
System.out.println('&');

OUTPUT :-

&
The char data type can only hold 1 character. So if you use 2 or more characters in the a char variable like this : 
char ch = '12';
        System.out.println(ch);

The compiler will throw an error similar to this :

Program.java:3: error: unclosed character literal
                char ch = 'X1';
                          ^

4. short

The short data type is quite similar to the byte data type.

It uses 2 bytes or 16 bits of the computer's memory and can store numbers from -32768 to +32767.

Here's how to use the short data type :

short k = 1501;
        short d = 1103;
        System.out.println(d);
        System.out.println(k);

OUTPUT :-

1103
1501
Again don't forget to use short for a number that is less than -32768 or greater than +32767.

5. int

The int data type is probably the most used data type for storing numerical values*.

* Which is non-floating point.

It can store an extensively wide range of numbers from -2147483648 to +2147483647.

One int variable occupies 4 bytes of the computer's memory.

Here's how you can declare an int variable :

int i = 15011103;
        System.out.println(i);

OUTPUT :-

15011103
Be careful to store only non-floating point values in data types like int, short & byte. Storing a floating-point value in any of those like this : 
int x = 1501.101;
        System.out.println(x);

will raise an error.

6. long

The long data type is an extended variant of the int data type.

It uses 8 bytes of the computer's memory and can store a number from -9223372036854775808 to +9223372036854775807.

We typically use this data type to hold immensely large numbers, for example a phone number*.

* Actually we can use String for that, but that's a separate topic and we will talk about it later.

Here's how you can declare a long variable :

long phone_number = 3712894560;
        System.out.println(phone_number);

OUTPUT :-

3712894560
Be careful to store only non-floating point values for this data type as well

7. float

The float data type can store floating point ( decimal ) values.

It uses 4 bytes of the computer's memory and can store a number from 1.4E-45 to 3.4028235E38.

Using the float data type, we can store floating-point values with 6-7 significant digits.

Here's how you can declare a float variable :

float f = 3.14f;
        System.out.println(f);

OUTPUT :-

3.14
The ‘f’ in 3.14f is optional. However, its a good practice to use the ‘f’ suffix for float data types, even though it is not printed in the output.

Here's the reason to use the ‘f’ suffix :-

float x = 0 has an implicit typecast* from int to float.
float x = 0.0f does not have such a typecast*.
float x = 0.0 has an implicit typecast* from double to float.


* Typecasting is explained in the next lesson.

8. double

The double data type can store floating point ( decimal ) values.

It uses 8 bytes of the computer's memory and can store a number from 4.9E-324 to 1.79E308.

Using the double data type, we can store floating-point values with 15-16 significant digits.

Here's how you can declare a double variable :

double d = 2.718281828459045d;
        System.out.println(d);

OUTPUT :-

2.718281828459045
The reason of using ‘d’ suffix in 2.718281828459045d is same as explained above.
