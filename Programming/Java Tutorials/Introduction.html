Introduction to Java

Java, an Object Oriented Programming language [ OOP ] was developed by James Gosling at Sun Microsystems, which is now acquired by Oracle. The language was originally called Oak, named after the oak trees outside the creator's office, but it was later discontinued and re-named as Java.

Java language allows writing of both, standalone applications [ those which can run offline ] as well as applets [ a Java program which can be embedded on a webpage ].

Why to choose Java?

Well, Java is a great language to start from and I personally recommend you to start with Java as your first programming language, because, once you have mastered Java, you can go down to languages such as C and they will seem quite familiar [ as compared to someone who has directly started with C ], alternatively you can also go for a more popular, easier, and much more syntax-friendly language like python.

Aside from this, there are a few more striking key features of Java, like

  • Object Oriented
  • It is both compiled and interpreted
  • Because of the above point it is also platform-independent

which contribute to it's greatness. Also

  • It does not require a preprocessor (#) or the necessity to include header files unlike C and C++.
  • It is a case-sensitive language.
  • It is a robust language since it gives a huge emphasis on memory management.

Why is it a big deal?

The reason behind Java's modern day popularity and wide usage lies in it's potential to be platform-independent.

Java guarentees we will be able to *WORA

*'Write Once, Run Anywhere or sometimes called Write Once, Run Everywhere. It was a slogan created in 1995 by Sun Microsystems to illustrate the cross platform benefits of the Java language.'
~source Wikipedia.

This means that the binary can run on absolutely any Operating System that has Java installed on it. This is a really powerful advantage of Java. So now we don't need to worry about getting the binary compiled on every machine.

How platform-independency works?

To understand this we first need to understand how do the compilers and interpreters work.
Also we must note a few points :-
  • A human readable program is not understood by the computers.
  • The computer only understands binary [ the zeros and ones ].
  • The code which we will write in Java will be called source code.
  • Java is a high-level language so our code will need to be translated to low-level language [ like assembly ]
  • It can be done so by using a compiler and/or interpreter.
  • The machine code/binary code [ final stage of compiling, lowest possible code ] is platform-specific. Which means that the compiled machine code for windows cannot be used on a linux machine

The Compiler

A compiler is a special, complicated computer program which is created with the intention to help in converting high-level languages to low-level languages.

The compiler scans the whole program at once and translates it to *machine code.

* 'In computer programming, machine code, consisting of machine language instructions, is a low-level programming language used to directly control a computer's central processing unit (CPU). Each instruction causes the CPU to perform a very specific task, such as a load, a store, a jump, or an arithmetic logic unit (ALU) operation on one or more units of data in the CPU's registers or memory.
~source Wikipedia.

The Interpreter

The interpreter is similar to the compiler in numerous ways. Like the compiler, the interpreter is also used in converting high-level languages to low-level languages.

note : both, the compiler and the interpreter are unique for every language. this means that for example, a compiler made for java cannot be used to compile c language programs or vice-versa.
Using a compiler to compile a language which is meant to be interpreted is probably the worst idea to think of.

Compiler Interpreter
Scans the complete program at once and converts it into machine code. Translates the program line by line.
It displays all the errors at once. If any error found, the translation will stop right there. After the error is fixed, only then can the interpreter proceed further.
The overall execution is comparatively tremendous. The overall execution becomes slow.
Languages like C, C++, Go & Rust use a compiler. Languages like Perl, PHP, Python, Ruby use an interpreter.

Generally, programming languages are either compiled or interpreted. However, Java is both compiled as well as interpreted.

Now let's understand, how Java makes use of the compiler and the interpreter to make itself completely platform-independent.

The source code of a Java program is stored in a file with the extension .java. In order to run the program, we need to compile it first. So on compiling, the system generates another file with the extension .class. The .java file is human readable but not the .class file. Java uses the compiler to compile the source code. However, now there is a slight change, instead of translating the source code to machine code, the Java compiler converts the source code to an intermediate machine code which is common to all, i.e it is platform independent and it is called the byte-code. Despite being platform-independent, the byte-code cannot be executed since it is only compiled half way and the remaining part is left to compile. Java now attempts to use it's interpreter to further translate this byte-code. Java's interpreter is called JVM [ Java Virtual Machine ]. But wait... didn't we just say that the interpreter is also a software? Then how come the interpreter is called Java Virtual Machine? It is so because, the JVM will now act like it's host machine and convert the byte-code to executable machine code in accordance with itself. For example, if JVM is installed on a windows computer then it will interpret the byte-code for a windows machine and if it is installed on a linux computer then it will interpret the byte code for a linux machine. So it is ideal to call the interpreter, a machine. Moreover this way, the compiler and the interpreter work together to make the language platform-independent.

To summerize it, the Java compiler compiles the source code half way through and leaves it in a platform-independent stage in the form of byte-code. The Java interpreter or the JVM now takes it's role and completes the remaining task by translating the byte-code to machine-code according to the machine on which it is running.

Object Oriented Programming

An object is the most basic unit of Object Oriented Programming [ or OOP in short ]. In OOP, the complete task is simplified by spilitting it into multiple objects.
Just how a bridge is used to transfer stuff from one end to the other, similarly in programming, these objects act like a bridge to allow for the easy passage of flow of data between *functions.

*Functions are essentially a group of organised instructions used to perform a very specific task.

Procedure Oriented Programming

Procedure Oriented Programming is a programming style in which stress is laid on procedures, i.e. functions and not data.

The key features of POP [ Procedure Oriented Programming ] are :

  • Emphasis is on functions and not on data.
  • Data values often keep floating from one function to another, because of this the programmer may need to make necessay changes so as to NOT allow the program to deviate from it's purpose.
  • Traditional procedural languages use the *top-down programming approach.

*Top-down approach is a programming pattern in which complex structures are defined first and then they are divided into successive smaller pieces.

Object Oriented Programming

It is a systematic approach to programming which is based on the concept of "objects" which can contain data and code [ in the form of functions ].

Key characteristics of OOP are :

  • It is extremely applicable in solving complex problems.
  • It uses objects as a bridge to have data flow between functions.
  • It makes the complete task easier by simplifing it [ splitting it ] into multiple objects.
  • They often use bottom-up approach [ inverse of what is explained earlier ].

So a very brief distinguishing table between OOP and POP is given ahead

OOP POP
It is object oriented. It is procedure [ function ] oriented.
The complete program is divided into muliple objects. The complete program is divided into several functions.
It uses bottom-up approach. It uses top-down approach.
The concept of inheritance exists. Inheritance is not allowed.
For example Java, Python, Ruby, Rust, C++ For example C, Pascal, assembly.

Concepts of OOP

So there are 4 basic pillars or concepts which OOP uses. They are :

  • Data abstraction
  • Encapsulation
  • Inheritance
  • Polymorphism

Data abstraction

Suppose you have a computer. In order to work on it, you must boot it, and for that, you press the power button and once your Operating System starts, you can start working on it. This is all you need to know, but obviously there is more to it. You are not aware how and which circuits are fired up on pressing the power button and how the BIOS loads the bootloader which starts the kernel and later the Operating System. You are also not aware of how the kernel establishes the pathway between the software and the hardware world with the drivers. But does this mean that these things do not exist?
→ Obviously not! These things do exist, it's just that you are unaware of it. So this is a real world example of data abstraction.

Thus data abstraction is the process of hiding certain unncessary details and showing essential information to the user. In Java, abstraction can be achieved by either *abstract classes or *interfaces.

*All these concepts are very advanced so you will learn this later in the series.

Encapsulation

Encapsulation is the mechanism of compiling data and code [ functions ] into a single unit.

Encapsulation

As explained in the above image, the data and functions are wrapped together in a single unit. With the help of encapsulation, functions cannot access data of other units [ or more specifically objects ]. Functions can only communicate with other functions and not the data, thus in order to use the private data of the other objects, they call the functions of those objects which help in fulfilling their purpose however, by no means can a function access the private data of another object.

Encapsulation is very beneficial because

  • the functions maintain the privacy of the data of other objects.

Inheritance

A class is a template or blueprint for creating objects. It defines the data and functions necessary for creation of objects.

Let's take a real world example of the cars. Here the class will be 'Vehicles'. It can be further classified into 'Motor' and 'Non-motor' vehicles. Each have certain properties [ or more techinically called attributes ].

The complete structure is given below

Vehicles
  • Motor vehicle
  • Engine Capacity
  • Colour
  • Non-motor vehicle
  • Tyre type
  • Colour

We know that both, motor and non-motor vehicles inherit some very specific properties from the class 'Vehicles' like, both of them have tyres and a metal frame.

Thus, Inheritance is the mechanism to link some common properties between two or more classes.

So, the class that gets inherited to another class is called Super Class and the class that inherits from the super class is called sub class.

note : the properties of the super class are when derived by the sub class, they may be used to perform another task in the latter. This concept is called reusability. Thus a very simple defination should be
Reusability is the concept by which the properties of the super class are shared by the sub class to perform another task.

Polymorphism

The term polymorphism literally means 'several forms'. Just how same words have multiple meanings similarly in Java the same function name can be used to satisfy multiple purposes.

For example, if naming the functions would have been like

  • 'volumeCylinder'
  • 'volumeCone'
  • 'volumeSphere'

then it would have been difficult for the programmer because (s)he would have to remember a lot of function names. To ease out this process, a concept of Function Overloading was introduced.

Function oveloading means to use the same name for multiple functions.

Thus, Java implements Polymorphism through Function Overloading.

So a proper defination for polymorphism would be
It is the concept of using the same function name to carry out multiple purpose.
