What is a buffer overflow?

So you might be wondering, what actually is a buffer overflow. Well, a buffer is a section in memory allocated for the storage of data in future. When excess data [ more than what was expected to be stored ] is written into the buffer, it causes the excess data to be leaked/overflowed to the adjacent memory locations [ assuming ALSR to be disabled, you will learn about it in future tutorials ].

Why is it a big deal?

The attacker can use a program which depends on user input by taking input through vulnerable functions like gets, read, strcpy, etc. The input which the attacker feeds to the program may contain memory address to change code execution or opcodes for instructions which ultimately help the attacker in changing the original purpose of the program.

Classification of buffer overflow

A buffer overflow is generally of two types:

  • Stack-Based

  • Heap-Based

Heap-based buffer overflows are particularly difficult to target & exploit. So for the sake of simplicity, we will be covering the Stack-based buffer overflow.
